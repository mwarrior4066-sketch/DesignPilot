# Precedence Model

DesignPilot uses a layered instruction model:
1. stable governance
2. routing metadata
3. task contracts
4. profile and skill logic
5. task-specific dynamic logic

## Conflict rules
- canonical source beats runtime overlay
- the governing route owns the evidence burden
- fail closed when ambiguity affects proof honesty, feasibility, or safety
- supporting skills sharpen the answer but do not replace the governing contract
