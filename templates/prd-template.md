# Product Requirements Document (PRD)

## Required fields
- Title
- Status
- Owner
- Stakeholders
- Problem statement
- User and segment
- Functional requirements
- Success metrics
- Risks and dependencies

## Field standards
### Problem statement
- Good looks like: names the workflow failure, affected user, and business consequence
- Minimum detail: 3-5 sentences with a clear before state and target after state
- Failure mode: describing only the feature request, not the underlying problem

### Success metrics
For each metric include:
- metric type
- baseline
- target threshold
- measurement method
- decision window
- failure condition
- owner

### Risks and dependencies
- name the hidden back-end, data, compliance, or workflow dependencies
- state what must be true before implementation starts

## Anti-patterns
- “Increase engagement” with no threshold
- shipping scope with no owner
- copying roadmap bullets into the PRD without decision consequences
