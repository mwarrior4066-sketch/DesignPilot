# User Persona

## Required fields
- Name or segment label
- Role and context
- Main job to be done
- Repeated pain points
- Trust barriers
- Common tasks
- Evidence source

## Field standards
### Role and context
- Good looks like: role plus real workflow conditions, not just a job title
- Minimum detail: one paragraph naming environment, constraints, and tool maturity
- Failure mode: a fictional-sounding archetype with no proof source

### Trust barriers
- name what makes this user doubt a solution
- include one sign they would look for before trusting it

### Evidence source
- include interview, benchmark, market research, support log, or project observation

## Anti-patterns
- persona written as a mood board
- “busy professional” with no actual workflow detail
- no evidence source
