Why pass-05, 06, 07 are persistent failure points
Root Cause Analysis
These three tests have the highest failure rates across all scored batches (31%, 25%, 46% respectively). The analysis below traces each root cause, fix applied, and remaining exposure.
pass-05 — pdf_remediation_plan
Root cause: forbidden shortcut negation check missed markdown formatting
Models correctly write do not flatten the PDF in their Risk Controls section — exactly the right response. The validator's positively_present() function checked for negation prefixes before the phrase, but the window content was do **not** with markdown bold markers. The regex matched literal "do not" not "do **not**", so the check failed and the validator hard-failed the output for recommending the very thing it was explicitly rejecting.

Fix: positively_present() now strips * and _ markdown characters and normalizes whitespace before running the negation check. Window widened from 35→80 chars. Negation vocabulary expanded to include rejected, prohibited, forbidden, warning, caution. All five affected cases now pass.
Remaining risk: missing_rationale on section-headed outputs
Some outputs structure rationale as sub-sections with bold headings rather than inline causal language. These now pass structural rationale check but may still soft-fail attached rationale. This is a soft issue (no hard fail) and acceptable.
pass-06 — brand_positioning_pass
Root cause: overclaim "leading" fired on directional usage
The no_market_superiority overclaim rule forbids "leading" to prevent "market leading" claims without proof. But outputs frequently use "leading with a freelancer-first approach" — a directional verb, not a superiority claim. The rule had no exception for directional uses.

Fix: Added leading with, leading to, leading by, by leading, niche, category to allowed_if_any. The rule now only fires when "leading" appears without any of these signals.
Root cause: implementation_constraint evidence missing brand vocabulary
The implementation_constraint evidence class token list covered technical tokens (constraint, token, boundary) but not brand constraint vocabulary. Brand outputs say "avoid", "convention", "category trust", "regulatory" — none of which were in the list.

Fix: +13 tokens including avoid, convention, trust, regulatory, tension, risk.
pass-07 — case_study_rewrite: 46% failure rate — partially genuine
Three distinct failure causes, two fixed, one genuinely a model quality gap:

1. Overclaim "validated" (fixed): no_external_validation_claim rule triggered on "validated the effectiveness of the design" — which is proof language, not an external validation claim. Added validated the, validated through, user testing, usability test, measurable to allowed_if_any.

2. honesty_tradeoff decision tokens (fixed): The decision required words like "tradeoff", "honesty", "smoothness" but case study rewrites use narrative signals. Added +11 tokens: however, while, although, limitation, challenge, constraint, key learning, next iteration.

3. missing_tradeoff (genuine — prompt fix applied): Case study rewrites produce polished narrative prose that omits design tradeoffs entirely — they are not mentioned in the original text and not requested by the prompt. Three providers (Gemini, OpenAI, xAI) consistently fail this gate. The prompt now explicitly requests: "include one explicit design constraint or tradeoff you had to navigate." This fix affects future runs; existing batch-0008 outputs remain partially failing.
