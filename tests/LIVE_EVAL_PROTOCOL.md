# Live Eval Protocol

Use this protocol for blinded comparisons between the pack and a baseline.

## Requirements
- same model
- same prompt
- same constraints
- randomized A/B order
- human scoring without pack identity revealed
- stored scores and notes by pack version

## Minimum run set
- 8 to 10 representative prompts
- at least one screenshot-led critique
- at least one architecture task
- at least one proof-sensitive writing task
- at least one PDF or document-semantics task

## Reporting
Store:
- prompt id
- pack version
- evaluator id or label
- rubric scores
- overall preference
- evidence mode: live or proxy
- run date
