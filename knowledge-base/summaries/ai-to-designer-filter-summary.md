# AI-to-Designer Filter Summary

## Purpose
Condensed rules for transforming routed expert output into designer-readable explanation.

## Use when
- the routed answer is technically correct but too dense
- the active explanation tier is Functional or Integrative
- the user needs both truth and actionability

## Core shape
- core verdict
- impact or context
- constraints and implementation realism
- next steps

## Preserve
- governing route
- required decisions
- evidence burden
- implementation truth
- proof honesty

## Adapt
- jargon density
- explanation depth
- examples
- framing
- next-step specificity
