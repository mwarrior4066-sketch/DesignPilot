# Front-End Implementation Review Template

## Architectural framing
- primary user task
- rendering model under review
- visible performance or maintainability risk

## Boundary and state model
- server vs client boundaries
- state ownership map
- status or state-machine model
- semantic host elements and custom-widget exceptions

## Rendering and mutation strategy
- rendering path
- data-fetching posture
- mutation handling and rollback behavior
- loading, error, and degraded UI rules

## Performance and accessibility risk
- hydration or bundle-cost note
- re-render hot spots
- keyboard/focus dependency
- token or implementation constraint

## Safer implementation path
- first change
- second change
- deferred or optional work
- verification notes
