# Text Humanization Summary

## Purpose
Condensed operating rules for revising prose so it reads like authored writing instead of machine-shaped output.

## Use when
- the user explicitly asks to humanize or naturalize text
- prose-heavy explanation sounds repetitive, mechanical, or over-scaffolded
- designer-facing narrative or rationale needs a realism pass after structural work

## Default rules
- preserve meaning before voice
- repair logic before rhythm
- reduce repeated sentence frames
- prune transitions that only simulate polish
- keep the writer’s actual tone instead of replacing it

## Do not use when
- the artifact is code, schema, validator output, or a strict spec
- the user needs literal structure more than prose texture
