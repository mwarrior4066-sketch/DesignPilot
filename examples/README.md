# Worked Examples

This folder contains production-grade worked examples, not thin stubs. Each example is built to do three jobs at once:
- show a human reviewer what good looks like
- act as a few-shot structural conditioning artifact
- link directly to a fixture and a validation result so release tooling can reject fake examples

## Required shape
Every example must include:
- input prompt
- selected route and loaded skills
- final output
- why the output passed
- what would have failed
- one revision pass that shows a weak draft being corrected

## Coverage
Current direct coverage includes:
- UI structure critique
- component spec
- dashboard audit
- backend feasibility review
- PDF remediation plan
- brand positioning pass
- case study rewrite
- accessibility feedback audit
- color system specification
- graphic critique
- layout reconstruction plan
- type system recommendation
- UX research gap map

- `real-brief-backend-feasibility-review.md` - real-use-case style backend feasibility review with explicit prototype and production claim boundaries.
